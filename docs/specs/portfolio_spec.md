# Portfolio Spec - Crypto Market Factor Lab

**Status:** v1.0 implementation contract  
**Primary artifact root:** `reports/portfolio_v2/`  
**Audience:** quant finance, crypto research, market-structure, and data/modeling reviewers

**v2.1 enhanced artifact root:** `reports/portfolio_v2_1/`

## Positioning

Crypto Market Factor Lab is a reproducible Python analytics framework for
BTC/ETH factor regimes, ETF-flow market plumbing, stablecoin liquidity,
cross-asset connectedness, and crypto-native market structure. It should be
presented as a portfolio-grade research engineering project, not as a
manuscript-first project.

## Portfolio Outputs

`python scripts/run_portfolio_pipeline.py` must generate:

- `reports/portfolio_v2/executive_summary.md`
- `reports/portfolio_v2/technical_report.md`
- `reports/portfolio_v2/data_atlas.md`
- `reports/portfolio_v2/manager_report.md`
- `reports/portfolio_v2/resume_bullets.md`
- `reports/portfolio_v2/model_cards/*.md`
- `reports/portfolio_v2/figures/*.png`
- `reports/portfolio_v2/tables/*.csv`

The pipeline must select the newest complete dated table/figure bundle. A newer
partial run must not displace the latest complete portfolio source bundle.

## Portfolio v2.1 Enhanced Outputs

`python scripts/run_portfolio_v2_1_pipeline.py` must generate a separate packet
under `reports/portfolio_v2_1/` without overwriting the stable v2 packet:

- executive, technical, analytics, data-atlas, resume, manifest, and diagnostic files
- model cards for static OLS, block partial R^2, ETF lead-lag, rolling correlations,
  stablecoin liquidity, BTC-native lab, structural breaks, VAR/FEVD, and events
- tables for full-vs-reduced block partial R^2, nested ablations, ETF lead-lag,
  rolling correlations, stablecoin liquidity, and BTC-native diagnostics
- PNG figures `F10` through `F52` plus copied baseline reference figures

## Narrative Rules

- Use association, exposure, contribution, market plumbing, lead-lag evidence,
  and regime diagnostics.
- Do not claim ETF flows caused BTC or ETH returns.
- Do not describe the current structural-break implementation as Bai-Perron.
- Do not describe rolling attribution as Shapley/Owen.
- Keep the frozen data panel through roughly April 2026 as the core release.

## Acceptance Criteria

- Portfolio artifacts exist and are regenerated by one command.
- `Data/` files are not modified.
- Existing dated research outputs remain intact.
- README first screen communicates a portfolio research system.
- Model cards disclose inputs, methods, interpretation limits, and risks.
- The manager report separates verified repo facts from inference and external
  research context.
