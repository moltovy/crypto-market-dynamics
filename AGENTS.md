# AGENTS.md — CryptoQuant Research Operating Constitution

This file is the **single source of truth** for any human or LLM agent working on this project. Read this file before making any edit. It is intentionally short; the long-form rationale lives in the referenced files.

---

## 1. Mission (non-negotiable)

Produce **one defensible empirical-finance working paper** on **how BTC and ETH factor exposures evolved around the January 2024 spot-ETF structural break**. The headline diagnostic is rolling OLS + block-level partial R². The formal break evidence is Bai-Perron + Chow on pre-registered event dates. Dynamic transmission is shown with a compact 4-variable VAR / FEVD system.

The canonical spec of the project is [`project_research_plan.md`](./project_research_plan.md).

---

## 2. Non-goals (do NOT do)

- Do not build predictive ML models or trading strategies (see [docs/context/00_project_context_and_goals.md](./docs/context/00_project_context_and_goals.md)).
- Do not treat Machine Learning as the headline method. ML is allowed only for *support* diagnostics (LASSO screening, boosted-tree importance). See [docs/context/03_quantitative_methods_and_analysis_menu.md](./docs/context/03_quantitative_methods_and_analysis_menu.md).
- Do not modify any file under `Data/` outside the curation pipeline at `tools/`.
- Do not invent citations. Any 2025-2026 arXiv link cited in drafts must be verified via Context7 or scholar.
- Do not exceed the do-not list in [docs/context/00_project_context_and_goals.md](./docs/context/00_project_context_and_goals.md).

## 5. Quality gates (every hand-off)

1. **Inputs read** — list file paths.
2. **Outputs written** — list file paths + row counts + hashes.
3. **Confidence score** — 0-100% with supporting evidence.
4. **Open questions** — hand-off-forward list.
5. **Next agent** — explicitly named.

---

## 6. Citation discipline

- Every empirical claim in markdown or paper drafts must cite a file path in `reports/` or `Data/` or `docs/`.
- Every external citation (arXiv, SSRN, journal) must have a verified DOI or link.
- The two memos [`reports/prior_ai_outputs/Beyond Correlation_ Quantifying Bitcoin's New Role in Financial Markets Through Structural Breaks, Flow Dynamics, and Systemic Risk.md`](./reports/prior_ai_outputs/) and [`reports/prior_ai_outputs/deep-research-report.md`](./reports/prior_ai_outputs/) contain unverified 2026 arXiv IDs — **do not reuse their citations** until the P0 audit in [§17 of the plan](./project_research_plan.md#17-execution-roadmap) is complete.

---

## 7. Defensive commits

- Checkpoint with a commit before any multi-file edit.
- Target ≤3 files per commit where feasible.
- Interface-freeze new modules after 1 consumer exists.
- Commit message style: `<type>(<scope>): <summary>` where `<type>` ∈ `{feat, fix, chore, refactor, docs, test, reorg}`.

---

## 8. Data integrity

- `Data/` is governed by the curation pipeline at `tools/`. Do not hand-edit CSVs.
- `Data/_meta/curation_log.md` is append-only.
- `Data/MASTER_DATA.{md,txt,csv}` is machine-generated by `tools/data_curation/06_build_inventory.py`. Never hand-edit.
- Any data change must be reproducible via `make curate`.

---

## 9. Environment

- Python 3.11+, managed by `uv`.
- All deps in [`pyproject.toml`](./pyproject.toml); lock committed as `uv.lock`.
- Never commit `.env`; use [`.env.example`](./.env.example).
- Notebooks use the `_template.ipynb` header-cell convention enforced by `nbstripout` + `.cursor/rules/04-notebook-hygiene.mdc`.

---

## 10. Escalation

If you hit any of the following, stop and ask the human:

- A tool script fails on actual data (not dry-run).
- A prior AI claim contradicts `docs/context/` or this file.
- A pre-registered break date moves.
- A core figure or table is about to be deleted.
- The 3-student timeline is at risk.
