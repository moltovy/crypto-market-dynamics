# Project Walkthrough

## 1. Source Data Inventory

Start with `Data/MASTER_DATA.csv` and `Data/MASTER_DATA.md`. These document the
curated multi-source CSV inventory.

## 2. Frozen Panel

The master panel lives under `reports/panels/`. Portfolio releases use the
frozen panel metadata and generated parquet without refreshing raw data.

## 3. Feature Construction

`src/cqresearch/features/panel.py` builds stationary features. Portfolio v2.1
adds realized-volatility features in `src/cqresearch/features/volatility.py`.

## 4. v2 Baseline

`reports/portfolio_v2/` packages the stable baseline: static OLS, rolling
drop-one R2, structural-break diagnostics, VAR/FEVD, events, data atlas, and
model cards.

## 5. v2.1 Analytics Expansion

`reports/portfolio_v2_1/` adds full-vs-reduced block partial R2, ablations,
lead-lag diagnostics, rolling correlations, stablecoin liquidity, and BTC-native
factor diagnostics.

## 6. Model Outputs

Tables live under `reports/portfolio_v2_1/tables/`. The manifest lists every
table, figure, report, and model card generated by the release pipeline.

## 7. Figure Generation

Figures live under `reports/portfolio_v2_1/figures/`, including `F10-F52` and
baseline references `F60-F62`.

## 8. Reports And Model Cards

Reader-facing reports live at the v2.1 root. Model cards live under
`reports/portfolio_v2_1/model_cards/`.

## 9. Testing

Unit tests cover calendars, config, imports, portfolio pipelines, partial R2,
lead-lag, ablation, volatility, and v2.1 manifest helpers.

## 10. Reproducibility

```powershell
uv run pytest
uv run mypy src/cqresearch
uv run python scripts/run_portfolio_v2_1_pipeline.py
```
