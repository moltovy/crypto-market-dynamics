# Project Context and Goals

## Team
- 3 MSc Finance students with a strong data analytics / technical background.
- Working with a professor.
- Goal is a serious, publishable, data-driven crypto / market-structure paper.
- The team is open to using frontier coding agents heavily, but wants decision-making grounded in professional standards from finance, engineering, research design, and software development.

## What We Do Not Want
- No naive "predict next-day price with ML" paper.
- No trading-strategy / alpha-claim framing.
- No black-box model as the core contribution.
- No vague macro storytelling without empirical structure.
- No overclaiming causality from noisy observational data.

## What We Do Want
- A strong quantitative / statistical / big-data analytics paper on crypto market evolution.
- A project that can be executed mainly through well-managed coding agents inside Cursor Pro/Codex/Antigravity 
- A workflow where major decisions are justified by research, current best practices, and measurable outputs.
- A path that lets the team explore broadly first, then narrow to the strongest, most defensible results.
- We also want you to give us input regarding other possible better ideas/project proposals based on the data we have, does not have to be what i gave you below, could be very micro/speciifc/niche related, since we have a lot of good data sources (polygon, defillama, artemis, cryptoquant, etc.) - think about anything bridging the gap between traidtional finance and crypto industry (defi, Cexs/Dexs, stablecoins, etc.) Ensure you analyze and understand all documentations revovling around artemis free terminal, defillama API ($300 subscription), Cryptoquant.com, and any other possible sources for free (Dune ANalytics, etc.) 

## Working Research Question
How did the relative importance of macro / cross-asset, institutional, crypto-liquidity, and native on-chain / ecosystem factors change for BTC and ETH in the post-ETF / institutionalized regime?

Equivalent framing:
Did BTC and ETH transition from being driven mainly by crypto-native internals toward being driven more by institutional flows and broader financial conditions?

## Current Preferred Paper Framing
This should be approached as a time-varying factor / market-structure paper, not an ETF-only paper and not an on-chain-only paper.

## High-Level Research Philosophy
- Use transparent econometrics first.
- Use ML only as a support tool for dimensionality reduction, clustering, variable screening, nonlinear diagnostics, and high-dimensional analytics.
- Treat BTC as the main asset and ETH as the structured comparison / robustness asset.
- Build broad datasets first, then filter down.
- Prefer daily data as the main panel, with weekly and monthly robustness.

## Practical Constraints
- Team is not building institutional HFT or market-making infrastructure.
- Time and complexity must stay bounded.
- Data sources have uneven start dates, so sample design must be modular rather than forcing one giant universal panel.
- Engineering should be strong, but not over-engineered.
- we also want your suggestions/alternative/differnt prespectives. No need to reinforce what we told you. BAse everything on the data, the research, and whats realistic/feasible given the most optimal cursor / agentic workflwo and set up and state of the art capabilities as of april 2026. 

## Current Possible Direction
A comparative BTC / ETH factor-evolution paper documenting whether the post-ETF regime changed the relative importance of:
- macro conditions,
- institutional flows,
- crypto-liquidity,
- and native on-chain / ecosystem drivers.
